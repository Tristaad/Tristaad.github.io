# _**Final Project Scope Statement**_

## _Project Overview_
This project will be a two-page personal portfolio website for Tristan Adame. The website will be designed as a student portfolio that introduces my background, my academic focus, and the skills I am developing through my Digital Media and Professional Communications coursework. It will include a homepage and a second page that give a simple introduction to my academic background, developing skills, and portfolio direction.

The project will not be a full professional career portfolio, since I do not have a large amount of career experience yet. Instead, it will focus on presenting myself as a student who is building skills in communication, digital media, basic web development, and project planning.

The website will be created as part of Project 03 and will be housed in a public GitHub repository. The final repository will include the website files, project planning documents, and a README file that explains the purpose of the project and links to important parts of the repository.

## _Project Purpose_
The purpose of this project is to create a simple and organized portfolio prototype that can introduce who I am, what I am studying, and what kinds of skills I am working to build. The project gives me a chance to practice planning a digital project before building it, while also showing basic HTML, CSS, Markdown, GitHub, and project management skills.  

This project solves the problem of needing one organized place to present my student background and developing skills. It will also help me practice treating a small website as a managed project instead of just a coding assignment.

## _Deliverables_
The completed project will include a single public GitHub repository with a clear file structure. The main deliverables will include the following items.
- A professional README.md file that explains the project purpose
- A live website link included in the README.md
- A link to the documentation folder included in the README.md
- An index.html file for the homepage
- An about.html file for the second page
- A style.css file used by both HTML pages
- A docs/ folder for project documentation
- A scope.md file with the revised project scope statement located in the docs/ folder
- A plan.md file with the project plan, risk analysis, and task schedule located in the docs/ folder
- A retrospective.md file with a reflection on the project process located in the docs/ folder

## _Technical Requirements_
The project will use HTML, CSS, Markdown, GitHub, and GitHub Pages or another approved method for publishing the website. The website will use a simple structure that matches the course requirements.

The root directory of the repository will contain these required files.
- README.md
- index.html
- about.html
- style.css

The repository will also contain this required folder.
- docs/

The docs/ folder will contain these required files.
- scope.md
- plan.md
- retrospective.md

The website will use valid basic HTML structure and one external CSS file. The CSS will apply consistent fonts, colors, spacing, and basic layout choices to both pages.

## _Out of Scope_
This project will not include advanced features beyond the course requirements. The website will be a simple two-page prototype, not a full professional portfolio website.  

The following items are out of scope for this project:
- No login system
- No database
- No contact form that sends messages
- No online store or payment system
- No JavaScript features unless specifically required later
- No large multi-page website beyond the required two pages
- No custom backend or server-side code
- No professional resume download
- No paid hosting or paid design tools
- No animations or moving parts that require extra code

## _Constraints_
#### **Time:**
The project must be completed within the course schedule and submitted by the assigned deadline. Because this project includes both a website and planning documents, the work will need to be divided into smaller tasks instead of being completed all at once. No extra time beyond the course schedule should be needed unless unexpected complications arise.

#### **Quality:**
The project must meet the minimum expectations listed in the Project 03 instructions and score well on the rubric. The website should be clear, readable, organized, and functional. The repository should also be professional enough that another person can understand the project purpose, file structure, and documentation. Everything will need to be completed while adhering to the templates, have a proper depth of analysis, have a realistic and achievable scope, and be properly formatted and submitted.

#### **Budget:**
The project will have no financial budget and will need no financial budget. It will use free tools and resources, including Visual Studio Code, GitHub, GitHub Desktop, and course materials.

## _Assumptions_
This project is based on the assumption that I will continue to have access to Visual Studio Code, GitHub, Canvas, and the course materials. It also assumes that the project requirements will stay close to the current Project 03 description.

I am assuming that a simple two-page student portfolio is an acceptable topic because the final project allows a personal portfolio as one of the website options. I am also assuming that the website does not need advanced features, since the assignment focuses on project planning, documentation, and a basic website prototype.

Finally, I am assuming that there will be no complications; whether technological, physical, or in my actual life, that would either shorten or postpone the time I can work on this project.

## _Success Criteria_
This project will be successful if the final repository contains all required files and folders, the website has two working pages, and both pages use the same external CSS file. The project will also be successful if the README.md clearly explains the purpose of the project and links to the live website and documentation folder.

The scope document will be successful if it clearly explains what the project is, what it will deliver, what is not included, and what limits or assumptions affect the work. The overall project should show that I can plan, organize, build, and reflect on a small digital project from start to finish.
