# Project Retrospective

## What Went Well

One thing that went well during this project was that I already had a starting foundation from earlier coursework. I had an existing GitHub repository, README.md file, index.html file, and style.css file, so I was not starting completely from scratch. This helped make the final project feel more manageable.

Another thing that went well was organizing the project around the checklist. Once I understood that the project needed a homepage, an about page, one shared stylesheet, a docs folder, and the three required Markdown files, it became easier to see what still needed to be done. The scope statement and project plan also helped keep the website realistic instead of letting it turn into something too large.

I also think the website became stronger once I separated the purpose of the homepage and the about page. The homepage works more as a portfolio landing page, while the about page gives more background about me, my academic path, my work style, and my future direction.

Another thing that went well was finding a way to keep working even when I could not comfortably work at my computer. I found and downloaded a GitHub compatible coding app on my phone and worked on a mobile branch so I would not mess up the main branch while using a different workflow. I am glad I did this because the app turned every save into a commit action, which would have flooded the main branch with a lot of descriptionless commits if I had worked there directly.

## What Went Wrong

One major challenge was that I lost a lot of planned work time. The entirety of Monday was spent building a chicken coop, so I was not able to work on the project that day at all. After that, my back was hurting from Tuesday to Thursday, which made it difficult to sit at my computer and continue working normally.

Another challenge was that I still had other things to take care of around the house while trying to finish the project. This included things like building a desk for my parents and picking up dog poo before the mowers mowed our yard. These were not directly related to the project, but they still affected the amount of time and energy I had available.

A project-specific challenge was figuring out how to make the two pages feel different without making them feel disconnected. At first, the homepage and about page were starting to repeat the same information. I had to think more carefully about what belonged on the homepage and what belonged on the about page.

Another issue was keeping the project simple. Since this is a portfolio, it was tempting to think about extra features or more advanced design ideas. However, adding too much would have made the project harder to finish and could have gone beyond the course requirements.

## What I Would Do Differently

If I did this project again, I would plan the purpose of each webpage earlier. I would decide from the beginning what the homepage should do and what the about page should do, instead of figuring that out later in the process.

I would also create the documentation files earlier and update them as I worked. This would make the scope statement, project plan, and retrospective feel more connected to the actual project process.

Another thing I would do differently is leave more extra time for unexpected problems. I did not expect to lose so much time because of the chicken coop, back pain, and other responsibilities around the house. Having more buffer time would have made the project less rushed.

I would also test the live website and links earlier. Checking the homepage, about page, GitHub profile link, README links, and docs folder link throughout the project would make the final testing step easier.

## Final Reflection

Overall, this project helped me practice treating a small website like a managed project instead of only a coding assignment. I had to think about scope, risks, required files, page content, documentation, and final testing. The project also forced me to adjust when my original work plan did not go perfectly.

Even though it became a bit of a rush to get everything done, I managed to accomplish the project in the end. I am proud of how polished and profrssional it looks, especially because the project had to be completed while dealing with other responsibilities and physical discomfort. The website is still a simple student portfolio, but it gives me a starting point that I can continue improving as I build more skills and complete more work in the future.
